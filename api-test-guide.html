🗺️ PIXPE Backend – API Test Guide

Base URL: http://localhost:3000  |  All protected routes require Authorization: Bearer <token>

Roles Available

admin · manager · surveyor · editor

Auth Type

JWT Bearer Token

File Uploads

multipart/form-data

JSON Requests

Content-Type: application/json

📑 Table of Contents

🔐 Auth — /auth
POST/auth/login Login with email + password 
{ "email": "admin@pixpe.com", "password": "admin123" }
{ "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
💡 Copy the access_token and use it as Bearer token for all protected routes.
POST/auth/otp/request Request Email OTP 
{ "email": "admin@pixpe.com" }
POST/auth/otp/verify Verify Email OTP 
{ "email": "admin@pixpe.com", "otp": "123456" }
POST/auth/admin/signup Create Initial Admin (Dev Only) 
{ "email": "admin@pixpe.com", "name": "Admin User", "password": "admin123", "phone": "9999999999", "role": "admin" }
⚠️ Role values: admin | manager | surveyor | editor
👤 Users — /users
POST/users adminmanager Create a new user 
Authorization: Bearer <token>
Content-Type: application/json
{ "email": "surveyor@pixpe.com", "name": "John Surveyor", "password": "pass1234", "phone": "9876543210", "role": "surveyor" }
GET/users?role=surveyor adminmanager List all users (optional role filter) 
role = admin | manager | surveyor | editor  (optional)
GET/users/profile Get my own profile (any role)
No body needed. JWT identifies the user.
GET/users/kyc-pending adminmanager Get all users with pending KYC
No body needed.
GET/users/kyc/status Get my KYC status (any role) 
{ "kyc_status": "PENDING", "is_verified": false, "reason": null }
GET/users/:id admin Get specific user by UUID 
:id = USER_UUID
PATCH/users/profile Update my own profile (any role) 
{ "name": "New Name", "email": "new@pixpe.com" }
PATCH/users/:id/role admin Change user role 
{ "role": "editor" }
role values: admin | manager | surveyor | editor
PATCH/users/:id/status adminmanager Activate or deactivate user 
{ "isActive": true }
POST/users/kyc Submit KYC document (any role) 
file = <KYC document file - image or PDF>
Send as form-data with key "file"
PATCH/users/:id/kyc-status adminmanager Approve or reject KYC 
{ "status": "APPROVED", "reason": "" }
// status: PENDING | APPROVED | REJECTED
DELETE/users/:id admin Soft delete user 
:id = USER_UUID
📍 AOI (Area of Interest) — /aoi
POST/aoi adminmanager Create a new AOI 
{
  "aoi_name": "Connaught Place Zone A",
  "boundary_geojson": {
    "type": "Polygon",
    "coordinates": [[[77.21,28.63],[77.22,28.63],[77.22,28.64],[77.21,28.64],[77.21,28.63]]]
  },
  "center_latitude": 28.6315,
  "center_longitude": 77.2167,
  "city": "New Delhi",
  "state": "Delhi",
  "pin_code": "110001",
  "priority": "HIGH",
  "deadline": "2025-12-31"
}
priority: LOW | MEDIUM | HIGH
GET/aoi adminmanager Get all AOIs
No body needed.
GET/aoi/assigned surveyoreditor Get AOIs assigned to me
JWT token determines which AOIs are returned.
GET/aoi/assigned/:id surveyoreditor Get one assigned AOI by ID 
:id = AOI_UUID
PATCH/aoi/:id/assign adminmanager Assign AOI to surveyor and/or editor 
{
  "surveyor_id": "UUID-OF-SURVEYOR",
  "editor_id": "UUID-OF-EDITOR"
}
PATCH/aoi/bulk-assign adminmanager Bulk assign multiple AOIs 
{
  "aoi_ids": ["AOI_UUID_1", "AOI_UUID_2"],
  "surveyor_id": "SURVEYOR_UUID"
}
PATCH/aoi/:id adminmanager Update AOI details 
{
  "aoi_name": "Updated Name",
  "boundary_geojson": { "type": "Polygon", "coordinates": [...] },
  "priority": "MEDIUM",
  "deadline": "2025-12-31",
  "notes": "Some notes"
}
PATCH/aoi/:id/start surveyor Start working on AOI → status: IN_PROGRESS
No body. Status changes to IN_PROGRESS.
PATCH/aoi/:id/submit surveyor Submit AOI → status: SUBMITTED
No body. Status changes to SUBMITTED.
PATCH/aoi/:id/close adminmanager Close AOI → status: CLOSED
No body. Status changes to CLOSED.
📷 Photos — /photos
POST/photos/upload surveyor Upload a photo to Supabase storage 
file         = <image file>
aoi_id       = UUID-OF-AOI
photo_type   = STOREFRONT | SIGNBOARD | INTERIOR | PRODUCT | CONTACT_DETAILS
latitude     = 28.6315   (optional)
longitude    = 77.2167   (optional)
GET/photos/my-uploads surveyor Get all my uploaded photos
No body. Returns photos uploaded by logged-in surveyor.
GET/photos?status=SUBMITTED adminmanager Get all photos (optional status filter) 
status = SUBMITTED | APPROVED | REJECTED | RESUBMITTED | ARCHIVED  (optional)
GET/photos/assigned editor Get photos assigned to me (editor)
No body.
GET/photos/assigned/:id editor Get one assigned photo by ID 
:id = PHOTO_UUID
GET/photos/aoi/:aoiId/editor/:editorId editoradminmanager Get photos by AOI and editor 
:aoiId    = AOI_UUID
:editorId = EDITOR_UUID
PATCH/photos/:id/assign adminmanager Assign photo to an editor 
{ "editor_id": "EDITOR_UUID" }
PATCH/photos/:id/status adminmanager Approve or reject a photo 
{
  "status": "APPROVED",
  "rejection_reason": ""
}
// status: SUBMITTED | APPROVED | REJECTED | RESUBMITTED | ARCHIVED
PATCH/photos/:id/resubmit surveyor Resubmit a rejected photo
No body. Sets status to RESUBMITTED.
PATCH/photos/:id/request-reupload editor Request surveyor to re-upload a photo 
{ "reason": "Image is blurry, please retake" }
DELETE/photos/:id surveyoradmin Delete a photo 
:id = PHOTO_UUID
